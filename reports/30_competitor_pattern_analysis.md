## Что переносим в V2 (как паттерн)
- Multi-transport модель с fallback между TCP/gRPC/xHTTP/HY2.
- Канарейка на уровне профилей (а не массовое мгновенное переключение).
- Split-routing политика:
  - `direct` для RU/локальных доменов и критичных сервисов;
  - `proxy` для остального трафика;
  - отдельные block-правила для нежелательных классов трафика.
- Health-aware выбор ноды и controlled rollout по процентам.

## Что строго запрещено переносить
- Любые чужие сетевые идентификаторы и креды:
  - IP/hostname,
  - UUID/user-id,
  - private/public keys,
  - passwords/tokens,
  - shortId/reality serverName/path.
- Любые буквальные конфиги competitor-файла.

## Решение для проекта
- Использовать только архитектурные принципы и статистические паттерны.
- Все идентификаторы/ключи/параметры транспорта генерировать заново в нашем контуре.
- V2 rollout вести через admin/test canary и только после backup+rollback gate.

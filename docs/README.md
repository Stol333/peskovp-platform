# Docs Scaffold
Каталог для технической и продуктовой документации.

Рекомендуемые разделы:
- `architecture/`
- `runbooks/`
- `api/`
- `operations/`
## Процесс обновления
- Любые изменения документации делать в отдельной ветке.
- Для изменений открывать PR в `master` с кратким описанием:
  - что обновлено;
  - почему это важно;
  - как проверить актуальность.
- Секреты, токены и приватные пути не добавлять в docs.

## Минимальный стандарт качества docs
- Практичность: только прикладные шаги и проверяемые инструкции.
- Актуальность: отражать текущую фазу и реальные артефакты.
- Трассируемость: при изменениях процесса обновлять релевантные файлы в `reports/`.
